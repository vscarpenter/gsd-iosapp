# Phase 5a — Sync Foundation (Design Spec)

> **Date:** 2026-05-31 · **Status:** spec drafted this session; design approved in brainstorming; **NOT yet planned or implemented**. Basis for the implementation plan. Next: write the plan (`docs/superpowers/plans/2026-05-30-phase-5a-sync-foundation.md`), then execute subagent-driven. First slice of **Phase 5 (Sync & Backend)**, which is split into **5a** pure core / **5b** auth + transport / **5c** sync engine / **5d** realtime + health + UI.

## 1. Purpose & scope
The **pure, backend-free core** of PocketBase sync (§7): the wire DTO, lenient wire-date handling, the bidirectional `Task`↔record mapper, last-write-wins comparison, the persisted sync queue, device identity, and spec-authored fixtures. All of it is deterministic and unit-testable with **no live backend** — the slice exists precisely so backend readiness never sits on the critical path. A new **`GSDSync`** package target is introduced here as the permanent home for all sync logic (transport arrives in 5b/5c).

The §8.4 cross-provider identity question is resolved as **(c) email-keyed** (convergence is backend-side; documented Apple "Hide My Email" private-relay caveat). This shapes 5b's auth UI, **not** 5a code — 5a is identity-agnostic (the mapper takes `owner` as a parameter).

**Out of scope (deferred):**
- Networking — REST client, OAuth (`ASWebAuthenticationSession`), Keychain token storage, token refresh → **5b** (§8).
- Pull/push engines, the single-flight coordinator, **enqueue-on-mutation wiring into `TaskStore`**, sync-history recording → **5c** (§7.4–7.7).
- SSE realtime, periodic safety net, health monitoring, sync-status UI / pull-to-refresh / history screen → **5c/5d** (§7.6–7.7).
- `owner`/auth resolution → **5b** (mapper takes it as a param).
- Fixtures are **spec-authored** (§7.1); reconciling them against real `api.vinny.io` responses is a **5b task**.

## 2. Builds on (Phases 0–4, on `main` at `phase-4-notifications`)
- **`GSDModel`** (zero deps): `Task` (full §5.1 field set incl. the device-local `notificationSent`/`lastNotificationAt`/`snoozedUntil`, the `parentTaskId` recurrence-lineage link, and `timeEntries`); `TimeEntry {id, startedAt, endedAt?, notes?}`; `RecurrenceType`; `Quadrant` (computed from `urgent`/`important`, never persisted into the struct). `Task` is already `Codable`/`Sendable`/`Equatable` with a lenient `init(from:)` (3c import tolerance).
- **`GSDStore`** (+GRDB): the `TaskRecord` mapper pattern (scalars direct, embedded collections as JSON via `GSDJSON`); `GSDJSON` — the **strict** millisecond ISO-8601 codec (rejects whole-second strings); `Migrations` (`registerV1`–`registerV3` + the `migrator` sequence); `AppGroupDefaults` (injectable suite + `Key` namespace, falls back to `.standard`); the repository pattern (`TaskRepository` async CRUD).
- **Package**: `GSDKit` with `GSDModel` + `GSDStore` library products; the app target depends on both via `project.yml`/XcodeGen.

## 3. Scope calls (design — approved in brainstorming)
- **New `GSDSync` target** — depends on **`GSDModel` only** in 5a; **GRDB-free and networking-free** (Foundation only). `Package.swift` gains the target, a `GSDSync` library product, and a `GSDSyncTests` test target with a `Fixtures` resource folder. The app's `project.yml` dependency on `GSDSync` is **deferred to 5c** (nothing in the app consumes it yet), so 5a's build surface is `swift test` + an unchanged-app smoke. `GSDSync` gains its `GSDStore` dependency in **5c** (the engine needs the repositories) — adding it now would pull GRDB into the pure layer prematurely.
- **`PocketBaseTaskRecord`** — a faithful §7.1 wire DTO: snake_case (`CodingKeys`), typed JSON arrays (`tags: [String]`; `subtasks: [Subtask]` — `GSDModel.Subtask` is already exactly the §7.1 `{id,title,completed}` shape, so it is **reused directly**; `dependencies: [String]`; `time_entries: [WireTimeEntry {id, startedAt(String), minutes}]` — a wire-specific type since the local `TimeEntry` carries `endedAt`/`notes` the wire form drops). **Date fields stay `String`** (raw `client_updated_at`, `client_created_at`, `due_date`, `completed_at`, `last_notification_at`, `snoozed_until`, and `WireTimeEntry.startedAt`) so leniency lives in one place. Carries the PocketBase `id` (recordId, distinct from `task_id`), `owner`, `device_id`. System `created`/`updated` are **omitted** (§7.1 forbids using them for sort/filter). **Absent values differ by type (§7.1):** date fields use the empty string (`""`→nil via `WireDate`), `notify_before`/`estimated_minutes` are JSON `null` (or key-absent) → nil via `decodeIfPresent`, and `time_spent` is a non-null number defaulting to `0`; the DTO decode is defensive across all three absent forms (empty string, JSON null, key absent).
- **`WireDate`** — the lenient counterpart to strict `GSDJSON`: `parse(String) -> Date?` accepts fractional **and** whole-second ISO-8601, maps `""` → `nil`, returns `nil` on unparseable input (feeding skip-malformed); `format(Date?) -> String` emits canonical fractional-seconds, `nil` → `""`. **Do not reuse `GSDJSON` for wire dates** — it is intentionally strict and would reject empty/whole-second wire values.
- **`TaskWireMapper`** (§7.2), three pure functions:
  - `toWire(_ task:, owner:, deviceId:, recordId:) -> PocketBaseTaskRecord` — flattens each `timeEntries {id,startedAt,endedAt,notes}` → `{id,startedAt,minutes}` (minutes = floored whole-minute duration `endedAt − startedAt`; a still-running entry with nil `endedAt` → `minutes = 0`), derives `quadrant` from the flags, formats dates via `WireDate`, maps `updatedAt`→`client_updated_at`, `createdAt`→`client_created_at`. `owner`/`deviceId`/`recordId` are **parameters**, never fetched — keeps the mapper pure (the 5c push engine supplies them).
  - `toDomain(_ record:, mergingInto local: Task?) -> Task` — one function, two modes. `local == nil`: full reconstruct — `timeEntries` rebuilt best-effort (synthesize `endedAt = startedAt + minutes`, `notes = nil`); `quadrant` **recomputed from `urgent`/`important`**, never trusting the wire value; empty wire dates → `nil`. `local != nil`: merge (below).
- **Merge rules** (`toDomain(_:mergingInto:)` when a local task exists) — these fields are **never taken from remote**, kept from the local copy:
  - `notificationSent`, `lastNotificationAt`, `snoozedUntil` — device-local per §7.4/§9.3.
  - `timeEntries` (and the **derived** `timeSpent`) — wire form is lossy (no `endedAt`/`notes`); §7.2 prefers local when both exist, so `timeSpent` (which `Task` computes from `timeEntries`) stays local too — taking it from remote independently would desync the count from the kept entries.
  - **`parentTaskId`** — §7.1 has **no wire column**; treated as device-local (preserve-on-pull, omit-on-push). Recurrence lineage stays correct per-device but does not propagate cross-device (**accepted trade-off**).
  - Everything else (title, `urgent`/`important`, `dueDate`, `completed`/`completedAt`, `recurrence`, `tags`, `subtasks`, `dependencies`, `notifyBefore`, `notificationEnabled`, `estimatedMinutes`) comes from remote.
  - On **push**, the device-local fields *are* written to their §7.1 columns (every device ignores them on pull, so the shared record harmlessly carries the last writer's values).
- **`LWW`** (§7.3) — compares **millisecond integers** (`Int(date.timeIntervalSince1970 * 1000)`, matching the web app): `resolve(localUpdatedAt: Date?, remoteClientUpdatedAt: Date?) -> LWWDecision` → `.takeRemote` / `.keepLocal` / `.noOp`; equal-ms or unparseable (nil) → `.noOp`. Guards both push and pull paths (the 5c engines call it).
- **`SyncQueueItem`** (§7.5 value type, in **`GSDStore`** so it can be persisted and the 5c engine can read it via the repository): `id`, `taskId`, `operation` (`.create`/`.update`/`.delete`), `timestamp` (Int ms), `retryCount`, `payload: Task?` (full task for create/update, nil for delete), `status` (`.pending`/`.failed`), `lastError: String?`, `lastAttemptAt: Int?` (ms), `failedAt: Int?` (ms).
- **`SyncQueueRecord`** + **`registerV4`** — a `syncQueue` GRDB table (mirrors `TaskRecord`/`GSDJSON`: scalars direct, `payload` as JSON text), registered after v3 in `migrator`.
- **`SyncQueueRepository`** — async CRUD: `enqueue(_:)`, `pending() -> [SyncQueueItem]` (ordered by `timestamp`), `update(_:)`, `remove(id:)`. **Built and tested, but intentionally NOT wired to `TaskStore` mutations** — that hook is 5c. This is the clean 5a/5c seam.
- **`DeviceIdentity`** (§7.8) — stable `deviceId` (UUID) + `deviceName`, generated once and persisted in App-Group `UserDefaults` (new `AppGroupDefaults.Key.deviceId`/`.deviceName`); the device-name source is **injected** so the package stays UIKit-free and the logic is testable with an injected defaults suite.

## 5. Testing
- **swift test (`GSDSyncTests`):** `WireDate` (fractional, whole-second, `""`→nil, garbage→nil; format round-trip + nil→`""`); DTO decode **defensive across absent-value forms** (`due_date: ""`, `notify_before: null`, and a key-absent variant); `TaskWireMapper.toWire` field-by-field (timeEntries flatten, quadrant from flags, date formatting); `toDomain` reconstruct (synthesize `endedAt`, recompute quadrant, empty-date→nil) and merge (each preserved field: device-local trio + `timeEntries`(+derived `timeSpent`) prefer-local + `parentTaskId`); **round-trip** `Task`→wire→`Task` asserting (a) the documented `endedAt`/`notes` loss and (b) `Task.id ↔ task_id` (the join key) preserved and kept **distinct** from the PocketBase record `id`; `LWW` (all four outcomes at ms granularity); **skip-malformed** driven by fixtures (a malformed record decodes to nil/throws and is skipped, not fatal).
- **swift test (`GSDStoreTests`):** `registerV4` creates `syncQueue` and coexists with v1–v3; `SyncQueueRepository` enqueue/pending(ordered)/update/remove round-trips incl. `Task` payload JSON and the delete-with-nil-payload case; `DeviceIdentity` returns the same persisted `deviceId` across calls with an injected defaults suite.
- **App:** build iPhone 17 + iPad M5 + simctl smoke — a **pure regression check** (the app is unchanged in 5a; nothing consumes `GSDSync` yet).
- **5b TODO (noted):** reconcile the spec-authored fixtures against real `api.vinny.io` responses once the backend is reachable.

## 6. Acceptance criteria (A42–A48)
- **A42** `PocketBaseTaskRecord` decodes/encodes the §7.1 wire shape (snake_case keys, typed JSON arrays incl. flattened `time_entries`) and is **defensive across the three absent-value forms** (`due_date: ""`, `notify_before: null`, key-absent); system `created`/`updated` omitted. The round-trip proves mapper **self-consistency, not wire-format fidelity** (the latter is confirmed against real responses in 5b). Unit-tested.
- **A43** `WireDate` parses fractional **and** whole-second ISO-8601, maps `""`→nil and garbage→nil; formats canonical fractional-seconds, nil→`""`. Unit-tested. (Strict `GSDJSON` is **not** used at the wire boundary.)
- **A44** `TaskWireMapper.toWire` flattens `timeEntries`→`{id,startedAt,minutes}`, derives `quadrant`, formats dates, maps `updatedAt`→`client_updated_at`/`createdAt`→`client_created_at`; `owner`/`deviceId`/`recordId` are parameters. Unit-tested.
- **A45** `toDomain(mergingInto:)` reconstructs (synthesize `endedAt`, recompute `quadrant`, empty-date→nil) when no local task exists, and merges preserving `notificationSent`/`lastNotificationAt`/`snoozedUntil`/`timeEntries`(+derived `timeSpent`)/`parentTaskId` when one does. Unit-tested incl. round-trip documented loss.
- **A46** `LWW.resolve` compares ms ints → `.takeRemote`/`.keepLocal`/`.noOp`; equal-ms or unparseable → `.noOp`. Unit-tested.
- **A47** The `syncQueue` table (v4) coexists with v1–v3; `SyncQueueRepository` enqueue/`pending()`(ordered)/update/remove round-trips incl. the `Task` payload JSON and nil-payload delete. **Built but intentionally unwired to `TaskStore` mutations** (5c). Unit-tested.
- **A48** `DeviceIdentity` returns a stable persisted `deviceId` (identical across calls) + a `deviceName`, via an injectable App-Group defaults suite + injected name source. Unit-tested.
- **Coverage:** `swift test` green for the `GSDSync` units + the `GSDStore` additions; both sims build; app smoke = unchanged-app regression.

## 7. Conventions (carried)
`GSDModel` stays zero-dep and **untouched** by 5a (no wire/PocketBase concepts leak into the domain core). `GSDStore` stays SwiftUI-free and `UserNotifications`-free. **`GSDSync` is GRDB-free and networking-free in 5a (Foundation only).** `GSDModel.Task` shadows Swift Concurrency's `Task` → qualify `_Concurrency.Task` where needed. Inject time / `Calendar` / defaults suite for determinism. The store stamps `updatedAt`. Swift Testing for logic; `xcodebuild` + simctl for the app; **`xcodegen generate`** after the `Package.swift`/`project.yml` change picks up new files. No `DEVELOPMENT_TEAM` in commits. Fixtures load via `Bundle.module` (`.process("Fixtures")` resource). Probe the date-leniency + LWW boundary math in `/tmp` before writing the plan.

## 8. Watch-outs
- **Wire-date leniency is THE subtle bit.** `GSDJSON` is strict and its own comment asserts "PocketBase always emits fractional seconds" — but §7.1 allows **empty** `due_date`/`completed_at`, and the assumption that fractional seconds are always present is not safe to bet correctness on. `WireDate` must be lenient (fractional + whole-second + empty→nil). Probe it.
- **`parentTaskId` has no §7.1 wire column** → device-local (preserve-on-pull / omit-on-push); recurrence lineage will not propagate cross-device (accepted in brainstorming).
- **`timeEntries` wire form is lossy** (no `endedAt`/`notes`) → prefer-local on pull; reconstruct best-effort (synthesize `endedAt`, `notes = nil`) only for new-from-remote tasks.
- **`timeSpent` is derived from `timeEntries`** (per `Task.swift`): on the prefer-local merge it must stay local — taking it from remote independently desyncs the count from the kept entries.
- **Absent values differ by type in §7.1** (dates → empty string, `notify_before`/`estimated_minutes` → JSON null): the DTO must decode empty-string, JSON null, **and** key-absent (defensive `decodeIfPresent`). This is the likeliest spot a spec-authored fixture silently diverges from the real backend — the 5b reconcile confirms it.
- **`SyncQueueRepository` is intentionally unwired in 5a.** Do not hook `TaskStore` mutations until 5c — a half-built push with no engine to drain the queue is exactly the "half-build sync" trap.
- **New SPM target + test resources.** The `Package.swift` change must declare the `Fixtures` resource (`.process("Fixtures")`) so `swift test` finds them via `Bundle.module`; run `xcodegen generate` so the project picks up the new target/files.
- **`GSDSync` must depend on `GSDModel` only in 5a** (never GRDB / GSDStore yet) — the `GSDStore` dependency arrives in 5c with the engine.
- **Identity is a 5b/backend concern.** 5a code is identity-agnostic (`toWire` takes `owner` as a parameter); the (c) email-keyed decision + the Apple private-relay caveat are documented for 5b, with no 5a action.
