# Phase 5c — Sync Engine (Design Spec)

> **Date:** 2026-05-31 · **Status:** design approved in brainstorming; **NOT yet planned or implemented**. Basis for the implementation plan. Next: write the plan (`docs/superpowers/plans/2026-05-30-phase-5c-sync-engine.md`), then execute subagent-driven. Third slice of **Phase 5 (Sync & Backend)** (5a + 5b done; **5c** sync engine; 5d realtime + cadence + UI + health).

## 1. Purpose & scope
Make tasks **flow both ways** — the payoff phase. A `SyncEngine` performs the bidirectional **active-task** sync: pull (§7.4), push via the queue (§7.5), a first-sign-in queue seed, deletion-reconcile, a single-flight coordinator, enqueue-on-mutation wiring into `TaskStore`, and a minimal set of triggers (launch / after-sign-in / manual "Sync Now"). Composes everything from 5a (mapper/LWW/queue/device-id) + 5b (client/auth). Pulled changes are written to the GRDB repo and reach the UI through `TaskStore`'s existing `ValueObservation` bridge (no UI poking, no circular dependency). The §8.4(c) email-keyed identity (validated in 5b) means a signed-in user's PocketBase records are the same set as the web app's.

**Out of scope (deferred to 5d):** SSE realtime (§7.6); the auto-cadence (every-2-min / on-foreground / on-network-regained); immediate debounced post-mutation push; the sync **status indicator** + pull-to-refresh (§7.7); the sync-**history table + screen** (§7.7); health monitoring (§7.7). **Archive/restore are device-local and do NOT sync** (brainstorming decision — §7.1 has no `archived` field; syncing it would mean a destructive server-side delete). GitHub/Apple sign-in remain later config (5b).

## 2. Builds on (Phases 0–5b, on `main` at `phase-5b-auth-transport`)
- **`GSDSync`** (currently Foundation-only, depends `GSDModel`): `PocketBaseTaskRecord`/`WireDate`/`TaskWireMapper` (incl. `toDomain(_:mergingInto:)` — the device-local-preserving pull merge) / `LWW.resolve` / `PocketBaseClient` (`authedRequest(path:method:token:body:)` for CRUD; `auth-methods`/`authWithOAuth2`/`authRefresh`) / `AuthService.validToken()` (proactive refresh → token, or nil) / `AuthConfig`. **5c adds a `GSDStore` dependency to the `GSDSync` target** (the engine reads/writes the repos) — clean layering (`GSDModel` ← `GSDStore` ← `GSDSync`; `GSDStore` never imports `GSDSync`).
- **`GSDStore`**: `SyncQueueItem`/`SyncQueueRepository` (5a — `enqueue`/`pending()` ordered/`update`/`remove`); `DeviceIdentity.current` (5a — `device_id`); `TaskRepository` (`upsert`/`fetchAll`/`fetch(id:)`/`delete(id:)`/`replaceAll`/`observeAll`); `TaskStore` (`@MainActor @Observable`; mutations all funnel through `repository.upsert`/`delete`; `tasks` fed by `observeAll()`; the two `// NOTE (Phase 5): enqueue a sync op here` markers in `archive`/`importTasks`); `AppGroupDefaults` (cursor storage).
- **App**: `GSDApp` (constructs/injects `store` + the 5b auth stack; `.task` launch hook); `SessionStore` (5b — signed-in state; the after-sign-in trigger point); `SettingsView` (Account section — add a "Sync Now" row).

## 3. Scope calls (design — approved in brainstorming)
- **`SyncEngine`** (`actor`, in `GSDSync`): deps = `PocketBaseClient`, `AuthService` (for `validToken()`), `TaskRepository`, `SyncQueueRepository`, `DeviceIdentity`, a cursor store (`lastSyncAt` in `AppGroupDefaults`), injected `clock`. **Writes pulled tasks via `TaskRepository.upsert` DIRECTLY** (preserving the wire's `client_updated_at` as local `updatedAt` — the LWW key; the engine never re-stamps and never enqueues, so pulled tasks aren't re-pushed). `owner` is the authed user id; `device_id` from `DeviceIdentity` (for the 5d echo-filter; harmless in 5c).
- **`sync(trigger:) async -> SyncResult`**, single-flight via an explicit `isSyncing` flag that **drops** a redundant concurrent trigger (coalesce, NOT actor-queue-and-run). Sequence:
  1. `token = validToken()`; nil ⇒ no-op (not signed in / offline).
  2. **First-sign-in seed** (when `lastSyncAt` is unset): enqueue every existing local **active** task as a push (`.update`). *The data-wipe guard* — protects pre-sign-in tasks (step 5's "not in queue") and uploads them.
  3. **Pull (upsert-only):** page through `GET /api/collections/tasks/records` filtered `owner = currentUserId && client_updated_at >= cursor−30s` (the `owner` API rule already auto-scopes; confirm the exact filter syntax at the live gate), **paging through `{items, page, totalPages}`** (data-completeness — don't drop page 2+); for each: skip malformed (lenient `PocketBaseTaskRecord` decode, §7.4 step 2); LWW vs local (`updatedAt`) → `toDomain(record, mergingInto: localIfAny)` → `repository.upsert`. No deletes.
  4. **Push:** bulk-fetch the remote index once (`task_id → {recordId, client_updated_at}`); for each `pending()` queue item due for an attempt: **LWW-guard** — for an upsert, skip if remote `client_updated_at` > the **payload task's `updatedAt`** (the value being pushed as `client_updated_at`; NOT the queue insertion `timestamp`, which is for ordering/retry only — see watch-outs); for a delete (no payload), skip if remote `client_updated_at` > the queue `timestamp`. Else upsert (`authedRequest` POST create if no recordId, else PATCH update by recordId) or delete; on success `remove` from queue; ~100 ms throttle; **429 → abort the loop**. **Retry is ACROSS-sync** (no blocking sleeps inside `sync()`): on failure bump `retryCount` + stamp `lastAttemptAt`, and skip the item until `now ≥ lastAttemptAt + backoff(retryCount)` (**5/10/30/60/300 s**); after **5** attempts mark `failed` (kept). With 5c's launch/manual triggers, retries fire on the next trigger (timely once 5d's cadence lands).
  5. **Deletion-reconcile (now safe):** over a FRESH remote index (post-push), delete local **active** tasks absent remotely **and** not present in the queue. (Archived table out of scope.)
  6. Advance `lastSyncAt` = max applied `client_updated_at`, clamped ≤ now, − 30 s. Return `SyncResult {pushed, pulled, conflicts, failed, error?}`.
- **Enqueue-on-mutation** (`TaskStore`, GSDStore): inject an optional `SyncQueueRepository` (defaulted to a **no-op**, mirroring `reminders: ReminderScheduling`) + a private `enqueue(taskId:operation:payload:)` helper. `create` → `.create`; `save`/`move`/`snooze`/timers/subtask ops/dependency ops → `.update`; `delete` → `.delete`; **`toggleComplete` → `.update` for the toggled task AND a separate `.create` for the spawned recurrence instance** (the spawn is a NEW local task). **Invariant: EVERY path that creates a local task enqueues** (`create` / recurrence-spawn → `.create`, import → upsert) — this is what stops deletion-reconcile from deleting locally-born tasks that never reached the queue. **`archive`/`restore` → NO enqueue** (device-local). `importTasks` **merge** → enqueue `.update` per written task; **replace** → enqueue `.update` per imported task (NOTE: replace's *remote deletion* of cleared-but-not-imported tasks is **not** handled in 5c — they re-appear on the next pull; documented limitation, revisit later). The repo's `delete` dependency-scrub cascade edits *other* tasks below the enqueue point → **heals via LWW on the next pull** (documented, not chased).
- **Cursor:** `lastSyncAt` — the max applied `client_updated_at`, stored as an ISO string (the wire format) in `AppGroupDefaults`; compared/filtered as ISO (lexicographic ISO-8601 ordering is correct), with the −30 s overlap computed as a `Date` then re-formatted via `WireDate`. **Cleared on sign-out** (`SessionStore.signOut` → engine resets the cursor) so the next sign-in re-seeds + full-pulls; **local tasks are NOT wiped on sign-out** (offline-first).
- **Triggers (5c):** sync-on-launch (in `GSDApp.task`, if signed in) · after-sign-in (`SessionStore.signIn` success → `engine.sync(.signIn)`) · manual **"Sync Now"** row in the Settings Account section. Both `SessionStore`/`GSDApp` and the engine are App-constructed (no circular dep). *Deferred to 5d:* the 2-min/foreground/network cadence + immediate post-mutation push.
- **Push request shapes:** create = `POST /api/collections/tasks/records` with the `toWire` body (owner + device_id set); update = `PATCH .../records/{recordId}`; delete = `DELETE .../records/{recordId}`. **Resolve the 5a carry-forward:** `notify_before`/`estimated_minutes` nil → confirm whether to send explicit `null` vs omit (the synthesized encoder omits; PocketBase likely treats absent = unchanged on PATCH — verify at the live gate; for create, send the full body).

## 6. Acceptance criteria (A57–A64)
- **A57** Pull pages through the real list shape (`{items, page, totalPages}` — page 2+ not dropped), skips malformed records, and LWW-upserts (`toDomain(mergingInto:)` preserves device-local fields). Unit-tested (2-page + malformed fixtures).
- **A58** **First-sign-in seed (data-wipe guard):** when `lastSyncAt` is unset, all existing local active tasks are enqueued before pull/reconcile; a task created while signed-out survives the first sync and is pushed. Unit-tested (the single most important property).
- **A59** Push: bulk remote-index; **LWW-guard compares remote `client_updated_at` against the payload task's `updatedAt`** (NOT the queue `timestamp`) for upserts (delete uses `timestamp`) — a stale-local + fresher-remote item (incl. a re-seeded one) is **skipped, not clobbering** the remote; create(no recordId)/update(by recordId)/delete; ~100 ms throttle; 429-abort; **across-sync** retry 5/10/30/60/300 s → `failed` after 5 (kept). Unit-tested incl. the stale-local-vs-fresher-remote skip.
- **A60** Enqueue-on-mutation: `TaskStore` create/update/delete enqueue via the injected (default-no-op) `SyncQueueRepository`; **`toggleComplete` enqueues the spawned recurrence instance as a separate `.create`** (every local-creation path enqueues); **archive/restore do NOT enqueue**; import-merge enqueues upserts. Unit-tested (recording fake, incl. the recurrence-spawn enqueue).
- **A61** Deletion-reconcile: over a fresh post-push remote index, deletes local **active** tasks absent-remotely **and** not-in-queue; archived table out of scope; runs after push. Unit-tested.
- **A62** `sync()` single-flight (`isSyncing` drops a concurrent trigger); orchestrates token→seed→pull→push→reconcile→cursor; returns `SyncResult`; advances the clamped −30 s cursor; offline/no-token → no-op. Unit-tested.
- **A63** Triggers wired: sync-on-launch (signed-in) + after-sign-in + manual "Sync Now" (Settings); `SyncEngine` constructed/injected in `GSDApp`; `GSDSync` target gains the `GSDStore` dependency. Build + simctl-render verified.
- **A64** **Live gate (manual, gates merge):** pull-first (web → phone) early; full bidirectional (phone → web via Sync Now; web → phone); signed-out-created task survives + uploads on first sign-in. **Not merged until this passes.**

## 7. Conventions (carried + new)
**`GSDSync` gains a `GSDStore` dependency in 5c** (the engine needs the repos + `DeviceIdentity`) — this is the one architectural change; `GSDStore` still never imports `GSDSync` (no cycle). `GSDStore` stays SwiftUI-free; the enqueue path uses only the `SyncQueueRepository` it already has. `SyncEngine` is an `actor`; single-flight is an explicit `isSyncing` **drop** (not actor reentrancy). The engine writes pulled tasks via the repository **directly** (preserves wire timestamps; never stamps; never enqueues). Inject `clock` + the request-executor + the repos for deterministic tests. **Fixtures grounded in REAL response shapes** (the paged `{items}` list MUST match the real backend — the 5a `decodeList`-was-a-bare-array lesson). `GSDModel.Task` shadows Swift's `Task` → `_Concurrency.Task`. `@MainActor @Observable` for any App state. Swift Testing for logic; `xcodebuild`+simctl for the App; `Package.swift` change (GSDSync→GSDStore dep) — no `project.yml`/`xcodegen` change (the App already depends on `GSDSync`). No `DEVELOPMENT_TEAM` in commits. Probe pagination + the cursor/clamp math in `/tmp` before the plan.

## 8. Watch-outs
- **THE one: first-sync data wipe.** Offline-first users have tasks before signing in; naive deletion-reconcile deletes them all (absent-remotely + not-queued). The **seed (step 2) MUST run before any deletion-reconcile** — A58 is its tested guard. Do not reorder.
- **Deletion-reconcile is destructive — three guards:** active-table-only, queue-aware (skip tasks pending/failed in the queue), and **after** push (so just-pushed tasks are in the fresh remote index). Runs last in `sync()`.
- **Data-loss sibling — seed must not clobber fresher remote (LWW-guard on payload, not queue-time):** the push guard compares remote `client_updated_at` against the **payload's `updatedAt`**, not the queue insertion `timestamp`. The seed enqueues old tasks with `timestamp ≈ now`; comparing against `timestamp` would push a stale local copy over a newer remote (cross-device loss). Compare against what's actually being written. (Refines §7.5's literal wording; honors §7.3's LWW-on-`client_updated_at`.) A58/A59 test it.
- **Data-loss sibling — every local-creation path MUST enqueue:** deletion-reconcile deletes local active tasks not in (remote ∪ queue). `toggleComplete`'s spawned recurrence instance is a new local task → enqueue it as `.create`, or the next sync deletes it. Audit create / recurrence-spawn / import. A60 tests the spawn enqueue.
- **Pagination is data-completeness, not cosmetic:** page through `{items}`/`totalPages` (or `perPage` high + verify) — else every task past page 1 silently never syncs. Confirm the real list shape at the live gate.
- **Single-flight = drop, not queue:** an in-flight `sync()` makes a new trigger a no-op; do NOT let an actor serialize-and-run a second pass.
- **The engine bypasses `TaskStore`:** it writes pulled tasks via the repository directly so it (a) preserves the wire `client_updated_at` (the LWW key — never re-stamp) and (b) does NOT enqueue (pulled tasks must not be re-pushed). Only user mutations (via `TaskStore`) enqueue.
- **Cursor:** 30 s overlap on read, clamp ≤ now − 30 s on advance; **clear on sign-out** (re-seed/full-pull next sign-in); never wipe local tasks on sign-out.
- **Push body null-vs-omit (5a carry-forward):** for create send the full `toWire` body; for PATCH update, confirm at the live gate whether nil `notify_before`/`estimated_minutes` should be explicit `null` vs omitted.
- **Import-replace remote-deletion deferred:** import-merge syncs (upserts); import-replace's removal of cleared tasks doesn't propagate remotely in 5c (they re-pull). Documented; revisit.
- **Restore + deletion-reconcile — narrow multi-device edge (OWNER DECISION at/before A64):** `restore` is device-local (no enqueue, per §8 above). **Single-device is safe** — `archive` never removes a task's queued op and `push` sends the enqueue-time payload, so a created-then-archived task still reaches remote, and a queued op only leaves via success (→ in remote) or failure (→ `.failed`, still in `allTaskIds()` → protected). **The residual:** if a task is archived here (stays in remote), then deleted remotely by another client, then **restored here**, the next deletion-reconcile removes it again (absent-remote ∧ not-queued ∧ active). Caught in the 5c independent review. **Decision for the owner:** make `restore` enqueue `.update` (one-line fix — resurrects the task remotely via LWW, likely the desired "I restored it, bring it back" behavior) vs keep device-local and accept the edge. Not auto-changed because it reverses the brainstorming archive/restore-are-device-local decision; resolve alongside the A64 live gate.
- **`device_id`/echo-filter:** set `device_id` on pushed records now (cheap); the realtime echo-filter that uses it is 5d.
