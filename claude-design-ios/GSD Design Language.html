The design reasoning

What changes, and why

The brief asked for refined-native — clearly iOS, just calmer and more editorial than the current build. Three deliberate moves get us there without fighting the platform.

01

Color earns its place

Today system-blue floods Browse, Dashboard, and Settings — it reads generic and competes with the matrix. We retire it. The only strong color in the app is the four quadrant pigments; chrome, nav icons, and charts go quiet graphite. Color now means something.

02

Type carries hierarchy

Apple New York (free, ships with iOS) does the headline and quadrant-title work; SF handles body and data. The serif is already on the matrix headers — we extend it consistently to page titles, sheets, and section heads so the whole app speaks one editorial voice.

03

Warm, not clinical

A warm paper neutral replaces flat iOS gray. It reads composed and unhurried — like good stationery — and makes the rationed pigments sit naturally. A cool ramp is provided too, so the team can flip the whole system with one token swap.

Typography

A serif for the voice, SF for the work

Headlines, page titles, and quadrant names are set in New York. Everything functional — body, labels, metadata, numerals — stays in SF for legibility and Dynamic Type fidelity. Never set body copy in the serif.

Onboarding — the rules

  • When: shown once on first launch, before any data exists. Mirrors the web app's first-visit About flow.
  • Four screens, always skippable. A Skip in the top-right of every screen jumps straight to the matrix. No screen is a wall.
  • Order: Welcome → The Matrix → Capture shorthand → Privacy & sync. Page dots show progress; swipe or tap Next.
  • No forced sign-in. The last screen's primary action is Start using GSD (enters offline). Sign in to sync is the quiet secondary — never the default.
  • Re-showable from Settings → About → Show Onboarding Again.
  • Reduce Motion: screens cross-fade instead of sliding; no animated illustration.
  • Voice: plain, unhurried, second person. State value, not hype. The privacy line must be literally true about where data lives.

Empty states — the anatomy

  • Mark — one quiet SF Symbol in ink-3 on a sunken tile (60pt). Tinted only to reassure (e.g. green check for “nothing overdue”). Never a colorful illustration.
  • Headline — one serif line (.title3), calm and factual.
  • Body — one sentence, ≤ 2 lines, ink-2. Say what will appear here and why it's empty.
  • Action — at most one, in tint. Omit it where there's nothing useful to do.
  • In-context empties (a quadrant in a non-empty matrix) use the dashed prompt row, not a full-screen takeover.
  • Reduce Motion / VoiceOver: no looping animation; the headline + body read as one announced label.

Archive · dimmed, read-only, reversible

Gesture map

  • Tap — open the editor (sheet on iPhone, inspector on iPad).
  • Leading swipe → — Complete / Uncomplete. Green fill, .success haptic, confetti on complete. Full-swipe past threshold triggers without lifting.
  • Trailing swipe ← — Snooze (slate) and Delete (rust). Delete asks once via a confirmation.
  • Long-press — lifts the card and opens the context menu: Edit · Complete · Start/Stop timer · Snooze · Duplicate · Share · Move to quadrant · Delete (destructive last, in rust).
  • Snooze & Move open a sub-menu of presets / the four quadrants.
  • iPad — the same menu on right-click / trackpad secondary-click; drag a card across a quadrant boundary to reclassify.
  • Reduce Motion — no confetti; the card cross-fades to its new state. Every action has a VoiceOver custom action.

Archive behavior

  • What it is — a separate store for completed tasks; each keeps every field plus archivedAt. Reached from Browse → Archive (sidebar item on iPad).
  • Read-only & dimmed — rows render at reduced opacity with a strikethrough title and the archived date. No inline editing.
  • Swipe → Restore (tint) re-inserts the task as active and clears archivedAt; Delete (rust) removes it permanently after a confirm. Both offer an undo.
  • Auto-archive — when enabled, completed tasks older than 30 / 60 / 90 days move here automatically; the sweep runs on launch and via background refresh.
  • Sync — archiving syncs as a delete of the active record; restoring as a create. “Archive Now” in Settings runs the sweep on demand.

Motion & interaction

Intentional, never decorative

Every motion has a job. All of it respects Reduce Motion — when it's on, transitions become simple cross-fades and confetti is suppressed entirely.

Completion

Tap or leading-swipe the check. The circle fills with the quadrant accent and draws a checkmark; a success haptic fires; the card eases to its dimmed, sorted position. Confetti (~120 + 2×60 particles) bursts from center — suppressed under Reduce Motion.

spring(response: 0.34, dampingFraction: 0.8) · haptic .success

Swipe actions

Leading → Complete (accent). Trailing ← reveals Snooze (slate) and Delete (rust). Rubber-band past threshold to trigger without lifting. Standard iOS row-swipe physics; haptic tick at the threshold.

interactiveSpring() · haptic .impact(.light) at threshold

Capture → classify

As shorthand is typed, the quadrant chip recolors and re-labels live (!! → Do First). On submit the field flashes the accent, the new card drops into its quadrant, and focus stays for rapid serial capture.

.transition(.move(.top).combined(with:.opacity))

Sheets & counts

The editor rises as a detented sheet (medium → large) over a dimmed board. Quadrant counts roll with a brief number transition when tasks move. Timer elapsed updates every second via TimelineView.

.presentationDetents([.medium, .large]) · .contentTransition(.numericText())

Copy-paste tokens

Drop-in SwiftUI values matching QuadrantStyle.swift and Theme.swift. Light / dark adaptive via the existing Color(light:dark:) initializer.

// Refined quadrant pigments — replace the values in QuadrantStyle.accent(_:)
static func accent(_ q: Quadrant) -> Color {
    switch q {
    case .urgentImportant:       Color(light: 0xB23A2E, dark: 0xE0705F) // Rust  · Do First
    case .notUrgentImportant:    Color(light: 0x2C6680, dark: 0x6FAACB) // Tide  · Schedule
    case .urgentNotImportant:    Color(light: 0x8A6A22, dark: 0xCFB266) // Ochre · Delegate
    case .notUrgentNotImportant: Color(light: 0x6F685F, dark: 0xA9A096) // Slate · Eliminate
    }
}

// Warm-paper neutrals — the system surface ramp
enum Surface {
    static let paper    = Color(light: 0xF4F1E9, dark: 0x17150F)
    static let sunken   = Color(light: 0xECE7DC, dark: 0x100E0A)
    static let surface  = Color(light: 0xFFFFFF, dark: 0x221E17)
    static let hairline = Color(light: 0xE3DDD0, dark: 0x322D24)
    static let ink      = Color(light: 0x211E1A, dark: 0xF1ECE2)
    static let ink2     = Color(light: 0x6E6760, dark: 0xA79F92)
    static let ink3     = Color(light: 0xA49B8D, dark: 0x6F685B)
    static let success  = Color(light: 0x3E7D52, dark: 0x6FB07F)
}

// Headlines — Apple New York, Dynamic-Type aware (Theme.swift already has this)
Text("Do First").font(.serif(.title3).weight(.semibold)).foregroundStyle(Surface.ink)
